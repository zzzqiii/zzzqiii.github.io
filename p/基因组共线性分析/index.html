基因组共线性分析

Jul 19, 2021

j首先要对两个基因组进行比对，可以选择用核酸序列也可以选择对蛋白序列进行比对，前者一般使用satsuma，后者使用mcmscan。核酸序列比对的缺点是不能排除重复序列的影响，但是因为satsuma使用起来很简单，所以还是了它来进行基因组共线性分析。

比对

 bsub -J satsuma -n 28 -o stm.log "/gpfs/bioinformatics/software/satsuma-code/SatsumaSynteny -q sinc_chr.fasta -t cmed_chr.fasta -o /gpfs/home/zuozhangqi/sinc_genome_data/11_Synteny/cmed/out_satsuma -n 28"

bioinfologics/satsuma2: FFT cross-correlation based synteny aligner, (re)designed to make full use of parallel computing (github.com)

800M的基因组大概要运行两三天的时间。

最后在输出目录out_satsuma中有一个satsuma_summary.chained.out的文件，提供了比对上区域。

可视化

satsuma的manual上提供的可视化工具是mizbee，但是摸索了半天还是不知道怎么用，遂放弃，还是用回circos。

cd ../data; ls -1
cmed_sinc.txt #告诉circos哪些区域需要连线
karyotype_cmed_sinc.txt #提供染色体核型的信息
cd ../etc/; ls -1
circos.conf
housekeeping.conf
ideogram.conf
link.conf
ticks.conf

satsuma_summary.chained.out中提取前6列，并在最后一列补充link的颜色信息，得到cmed_sinc.txt:

awk '{NF=6}1' satsuma_summary.chained.out > cmed_sinc.txt
sed -i "s/Chr/cmed_chr/g" cmed_sinc.txt
sed -i "s/ chr/ sinc_chr/g" cmed_sinc.txt
mv cmed_sinc.txt cmed_sinc_without_color.txt
python link.py cmed_sinc_without_color.txt > cmed_sinc.txt

生成karyotype_cmed_sinc.txt

sed -i "s/csup/cmed/g" karyotype.py
python karyotype.py sinc_chr.fasta cmed_chr.fasta karyotype_sinc.txt karyotype_cmed_sinc.txt

再circos 一下就好了。
