Jul 21, 2021

3d-DNA组装基因组的流程如下：

如果有了基因组的初步组装（draft assembly)，那么可以跳过第一步。

Hi-C & Juicer analysis

Juicer 能够将原始的测序数据转化为hic文件。

Home · aidenlab/juicer Wiki (github.com) 详细地说明了它的使用方法。

安装：

juicer有集群版本也有CPU版本，这两者的使用稍微有些不一样，之前在集群上用的时候没注意看，直接安装了CPU版本的，浪费时间白白跑了一个多月啥结果都没有，后来发现了这个问题，又重新安装了，却发现集群上的BWA用不了，于是放弃了，还是用来CPU版本的，不过在集群上跑应该会快很多。

git clone https://github.com/theaidenlab/juicer.git
cd 07_01_3d-dna-assembly-second
ln -s ~/juicer/CPU scripts  #如果是集群就要复制另一个目录
cd scripts/common
wget https://hicfiles.tc4ga.com/public/juicer/juicer_tools.1.9.9_jcuda.0.8.jar
ln -s juicer_tools.1.9.9_jcuda.0.8.jar  juicer_tools.jar
cd ../..
mkdir references
cp draft.fa references/   #复制基因组到references目录下
# this is optional, only needed for fragment-delimited files
mkdir restriction_sites
mkdir work; cd work #工作目录
mkdir fastq
mv <fastq_files> fastq/ #存放fastq文件
<myJuicerDir>/scripts/juicer.sh -D <myJuicerDir> #更改juicer.sh脚本中的目录

使用

在reference目录下，为基因组建立索引

bwa index draft.fa

根据基因组构建可能的酶切位点文件

generate_site_positions.py MboI draft draft.fa
awk 'BEGIN{OFS="\t"}{print $1, $NF}' draft_MboI.txt > draft.chrom.sizes

运行juicer

~/software/juicer-master/scripts/juicer.sh -g draft -s MboI -z draft.fa -y draft_MboI.txt -p draft.chrom.sizes

输出的结果文件在aligned目录下，merged_nodups.txt就是下一步3D-DNA的输入文件之一。

Candidate Assembly with 3D-DNA Pipeline

这个流程需要的输入文件为：

  1. a fasta file describing the draft assembly (.fasta, .fa and .fna files are allowed)
  2. a “merged_nodups.txt” file which is generated by the Juicer pipeline

运行

/home/zuozhangqi/software/3d-dna-master/run-asm-pipeline.sh draft.fa merged_nodups.txt

最后输出文件中，包含FINAL就是需要的结果。

把hic和assemebly文件导入到Juicerbox中，可以进行手工纠错。
