Mar 06, 2021
1 170

使用conda安装软件

很早就听说过conda安装软件很方便，今天终于在自己的电脑上尝试了。因为实验室的服务器没有联网，就在window子系统下试了一下。

下载

wget -c https://repo.continuum.io/miniconda/Miniconda3-latest-Linux-x86_64.sh

安装

chmod 777 Miniconda3-latest-Linux-x86_64.sh #给执行权限
bash Miniconda3-latest-Linux-x86_64.sh #运行

启动conda

cd /home/zuozhangqi/miniconda3/bin
chmod 777 activate 
. ./activate

当命令行前面出现(base)的时候说明现在已经在conda的环境中了。这时候输入conda list 命令就有反应了

添加镜像

conda config --add channels https://mirrors.tuna.tsinghua.edu.cn/anaconda/pkgs/free/
conda config --add channels https://mirrors.tuna.tsinghua.edu.cn/anaconda/pkgs/main/
conda config --add channels https://mirrors.tuna.tsinghua.edu.cn/anaconda/cloud/conda-forge/
conda config --add channels https://mirrors.tuna.tsinghua.edu.cn/anaconda/cloud/bioconda/

显示安装的频道

conda config --set show_channel_urls yes

查看已经添加的频道

conda config --get channels

编辑已经添加的频道

vim ~/.condarc

安装软件

conda install busco
which busco #查看软件安装位置
conda install 软件名=版本号 #安装特定版本软件
conda list #查看已安装软件
conda update 软件名  #更新指定软件
conda remove 软件名   #卸载指定软件

退出conda环境

. ./deactivate
#或者
conda deactivate

创建软件软连接

退出conda环境后之前安装的软件会用不了，这时只要把安装好的软件软连接到一个处在环境变量里的位置就可以使用了。

# 在/home目录下创建一个.soft文件夹
export PATH="~/.soft:$PATH"
ln -s ~/miniconda3/bin/busco ~/.soft

创建conda环境

之前创建的时候显示的是（base）这是conda的基本环境。有的时候可以给一下软件创建一个单独的环境。

conda env list  #查看当前存在的环境
conda create -n python2 python=2
# -n: 设置新的环境的名字
# python=2 指定新环境的python的版本，非必须参数
# 这里也可以用一个-y参数，可以直接跳过安装的确认过程。

conda会创建一个新的python2的环境，输入conda activate python2就可以启动这个环境。

退出环境

conda deactivate

删除环境

conda remove -n myenv --all

报错

Collecting package metadata (current_repodata.json): failed

CondaHTTPError: HTTP 000 CONNECTION FAILED for url <https://conda.anaconda.org/genomedk/linux-64/current_repodata.json>
Elapsed: -

好像是网络错误，刚开始Google了很多方法，还是报错，我还把按这个帖子的操作把~/.condarc中的default删掉了，以及把一个镜像的链接的https改成了http，不知道有没有效果。conda install busco后，一直有一条线在转，在conda的GitHub上看到了一个issue，跟我的情况一样：“Collecting package metadata” cannot proceed and never end。还参考了Solving environment: failed with initial frozen solve. Retrying with flexible solve的做法，还是一样报错。后来等到十点还是没好，我就放着不管回去了，结果第二天早上来发现装好了。有点迷。
