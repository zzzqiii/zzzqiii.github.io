基本应用结构

Jun 05, 2021

相似的页面使用模型去创建，创建一个article的应用，这个app里有一个Article模型，这个模型有标题和内容两个属性，同步到数据库。增加后台管理的article应用。

模型：

标题；内容；作者等。

python manage.py startapp  #应用名称

python manage.py startapp article

在mysite下多了一个article的文件夹。

修改该文件夹下的models.py:

from django.db import models

# Create your models here.
class Article(models.Model):  #继承models里已经定义好的Model类
	title = models.CharField(max_length=30)  #找到models里的字符串类型的字段，最长为30字符
	content = models.TextField()	#文本字段。
	#加入标题和内容两个属性

写入到models.py告诉django去使用这个app，在mysite文件夹下的setting.py里设置：

INSTALLED_APPS = [
    'django.contrib.admin',
    'django.contrib.auth',
    'django.contrib.contenttypes',
    'django.contrib.sessions',
    'django.contrib.messages',
    'django.contrib.staticfiles',
    'article',
]
#在这里增加前面新增的应用article

将修改同步到数据库。

python manage.py makemigrations
#在mysite目录下里多了article这个文件夹

python manage.py migrate

打开后台界面：

python manage.py runserver

编辑article目录下的admin.py文件

from django.contrib import admin
from . models import Article
# Register your models here.
admin.site.register(Article)

刷新后台界面可以看到多了一个article的版块：

点击增加文章，即可添加文章标题和内容。

后台界面可以设置成中文，在settings.py里修改LANGUAGE_CODE = ‘zh-Hans’

全局设置都在settings.py里修改。
